Detailed Description

The Hash<Key, Value, DataType, HashFunction> class provides multi-hash table.

It is a template class that can be customized with custom DataType and HashFunction. The supplied HashFunction must be implicitely convertible to uint64_t taking Key as its argument. The DataType must implement all methods that class adb::Data implements:

  • int64_t dataSize() const
  • Key key(int64_t index) const
  • const char *metaData(int64_t index, int64_t size) const
  • int64_t metaSize() const
  • void resize(int64_t dataSize, int64_t metaSize, char metaValue)
  • void setData(int64_t index, const Key &key, const Value &value)
  • void setMetaData(int64_t index, const std::vector<char> &values)
  • void setMetaValue(int64_t index, char value)
  • void setValue(int64_t index, const Value &value)
  • Value value(int64_t index) const

By allowing different DataType classes to be used as internal storage the Hash allows for example in-file storage.

The Hash provides dual API to be used as regular hash table or as a multi-hash table - multiple values associated to the same key.

Manipulating Data

To store data ensuring only one occurance of a Key will exist in the Hash use operator[]. The operator[] will insert new Key if it is not in the Hash. If you want to store new Key with a Value regardless of if there already is one or more in the Hash use insert(). To replace a Value with another use replace(). The replace() will not insert new Key if there was none however it will replace all occurances of Key or Key-Value pair. If you want to update only single Value use find() and iterator's iterator_base::operator*().

In order to remove all occurances of a Key or Key-Value pair from the Hash use remove(). If you want to remove only single instance use erase().

To remove all data from the Hash use clear().

Note: erase() never causes rehashing and returns iterator or const_iterator pointing to the next element after erased one. It is therefore safe to use while iterating over the Hash.

Retrieving Data

There are two principal ways to retrieve data from the Hash. One is using iterator and const_iterator in manual loop or ranged-based for. Alternatively you can access the elements using operator[], value() and values().

You may query the Hash for current number of stored elements with count(), whether it is empty with isEmpty() and the number of elements associated with a given key with appropriate overload of count(). The contains() overloads can be used to test whether the Hash has a given Key or Key-Value pair.

See also ADbHash.

adb::Hash::const_iterator Hash::erase(adb::Hash::const_iterator it)

Removes element pointed to by it from the Hash. Returns const_iterator pointing to the next element after the element being erased or end() if it pointed to the last valid element.

Note: This method never causes rehashing of the Hash.

adb::Hash::iterator Hash::erase(adb::Hash::iterator it)

Removes element pointed to by it from the Hash. Returns iterator pointing to the next element after the element being erased or end() if it pointed to the last valid element.

Note: This method never causes rehashing of the Hash.

adb::Hash::const_iterator Hash::find(const Key &key) const

Returns const_iterator to first element with key or cend() if the key is not in the Hash. If there are more occurances of key in the Hash their order is undefined.

adb::Hash::const_iterator Hash::find(const Key &key, const Value &value) const

Returns const_iterator to first element with key and value or cend() if the key with value is not in the Hash. If there are more occurances of key and value pairs in the Hash their order is undefined.

adb::Hash::iterator Hash::find(const Key &key)

Returns iterator to first element with key or end() if the key is not in the Hash. If there are more occurances of key in the Hash their order is undefined.

adb::Hash::iterator Hash::find(const Key &key, const Value &value)

Returns iterator to first element with key and value or end() if the key with value is not in the Hash. If there are more occurances of key and value pairs in the Hash their order is undefined.

adb::Hash::iterator Hash::insert(const Key &key, const Value &value)

Inserts new key with value as new element to the Hash. The Hash table might rehash. If there already is key this method will simply insert new one. If you want to replace a value with another use replace() (or find() and iterator_base::operator*()). If you want to insert new or replace existing value use operator[].

bool Hash::isEmpty() const

Returns true if there are no elements stored in the Hash.

int64_t Hash::remove(const Key &key)

Removes all occurances of key from the Hash. Returns number of removed elements or 0 if key was not in the Hash.

int64_t Hash::remove(const Key &key, const Value &value)

Removes all occurances of key with value from the Hash. Retturns number of removed elements or 0 if no key - value pair was not in the Hash.

int64_t Hash::replace(const Key &key, const Value &newValue)

Replaces all values associated with key with newValue. If there are no elements with key this method does nothing (it does not insert new element). Consider using overloaded method that takes old value as additional parameter if there are multiple values associated with the key. To replace only single value when there are multiple associated with key use find() and iterator_base::operator*(). Returns number of replaced elements (0 if no element was replaced).

int64_t Hash::replace(const Key &key, const Value &oldValue, const Value &newValue)

Replaces all values equal to key - oldValue pair with newValue. If there are no elements with key this method does nothing (it does not insert new element). Consider using overloaded method that takes old value as additional parameter if there are multiple values associated with the key. To replace only single value when there are multiple same with key - oldValue pairs use find() and iterator_base::operator*(). Returns number of replaced elements (0 if no element was replaced).

Value Hash::value(const Key &key, const Value &defaultValue = Value()) const

Returns Value associated with the key or defaultValue if there is no element with key. If there are more occurances of key in the Hash their order is undefined. Unlike operator[] you can specify your own defaultValue.

std::vector<Value> Hash::values(const Key &key) const

Returns vector of all values associated with key or empty list if the key is not in the Hash.

Reference<Value, DataType> Hash::operator[](const Key &key)

Returns adb::Reference object wrapping the hash's DataType and index pointing to the key's first occurance. If key is not in the Hash new one will be inserted possibly causing rehashing of the table. If there are multiple elements with key it is undefined which one will be chosen. Returned adb::Reference is assignable providing typical operator[] semantics:


  ADbHash<int, int> map;
  map[10] = 100; //inserts new element {10, 100}
  map[10] = 101; //replaces value associated with key 10

Value Hash::operator[](const Key &key) const

Returns call to value() with key.
